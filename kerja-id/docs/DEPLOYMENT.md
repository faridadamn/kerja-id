# KERJA.ID — Deployment Guide

## Prerequisites

- Node.js 20+
- Docker & Docker Compose
- PostgreSQL 16
- Redis 7
- Domain with DNS access

## Local Development

```bash
# 1. Clone repository
git clone https://github.com/kerja-id/kerja-id.git
cd kerja-id

# 2. Start infrastructure
cd infra && docker-compose up -d

# 3. Setup backend
cd ../backend
cp .env.example .env
# Edit .env with your settings
npm install
npx prisma generate
npx prisma migrate dev --name init
npm run prisma:seed
npm run start:dev

# 4. Setup frontend
cd ../frontend
cp .env.example .env
npm install
npm run dev

# 5. Access
# Frontend: http://localhost:3000
# Backend: http://localhost:3001
# Swagger: http://localhost:3001/api/docs
```

## Docker Deployment

```bash
# Build and run all services
cd infra
docker-compose up -d --build

# View logs
docker-compose logs -f backend

# Stop
docker-compose down
```

## Production Deployment (Kubernetes)

### 1. Build Docker Images

```bash
# Backend
docker build -t kerja-id-api:latest ./backend
docker tag kerja-id-api:latest registry.example.com/kerja-id-api:latest
docker push registry.example.com/kerja-id-api:latest

# Frontend
docker build -t kerja-id-web:latest ./frontend
docker tag kerja-id-web:latest registry.example.com/kerja-id-web:latest
docker push registry.example.com/kerja-id-web:latest
```

### 2. Kubernetes Manifests

```yaml
# backend-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: kerja-id-api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: kerja-id-api
  template:
    metadata:
      labels:
        app: kerja-id-api
    spec:
      containers:
        - name: api
          image: registry.example.com/kerja-id-api:latest
          ports:
            - containerPort: 3001
          env:
            - name: NODE_ENV
              value: "production"
            - name: DATABASE_URL
              valueFrom:
                secretKeyRef:
                  name: kerja-id-secrets
                  key: database-url
            - name: JWT_SECRET
              valueFrom:
                secretKeyRef:
                  name: kerja-id-secrets
                  key: jwt-secret
          resources:
            requests:
              memory: "256Mi"
              cpu: "250m"
            limits:
              memory: "512Mi"
              cpu: "500m"
          readinessProbe:
            httpGet:
              path: /api/v1/health
              port: 3001
            initialDelaySeconds: 10
            periodSeconds: 5
          livenessProbe:
            httpGet:
              path: /api/v1/health
              port: 3001
            initialDelaySeconds: 15
            periodSeconds: 10
---
apiVersion: v1
kind: Service
metadata:
  name: kerja-id-api
spec:
  selector:
    app: kerja-id-api
  ports:
    - port: 3001
      targetPort: 3001
```

### 3. Apply

```bash
kubectl apply -f k8s/
kubectl get pods
kubectl get services
```

## Environment Variables

### Backend

| Variable | Required | Description |
|----------|----------|-------------|
| `NODE_ENV` | Yes | `development` or `production` |
| `PORT` | Yes | Server port (default: 3001) |
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `REDIS_URL` | Yes | Redis connection string |
| `JWT_SECRET` | Yes | JWT signing secret (min 32 chars) |
| `JWT_EXPIRES_IN` | No | Access token expiry (default: 15m) |
| `JWT_REFRESH_SECRET` | Yes | Refresh token secret |
| `JWT_REFRESH_EXPIRES_IN` | No | Refresh token expiry (default: 7d) |
| `CORS_ORIGIN` | Yes | Allowed CORS origin |
| `SENDGRID_API_KEY` | No | For email notifications |
| `AWS_ACCESS_KEY_ID` | No | For S3 file storage |
| `AWS_SECRET_ACCESS_KEY` | No | For S3 file storage |
| `AWS_S3_BUCKET` | No | S3 bucket name |

### Frontend

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_API_URL` | Yes | Backend API URL |

## Database Migrations

```bash
# Create migration
npx prisma migrate dev --name description

# Apply in production
npx prisma migrate deploy

# Reset database (dev only)
npx prisma migrate reset
```
